## 3.3. Restrições Arquiteturais

As restrições impostas ao projeto que afetam sua arquitetura são:

-   O backend deverá ser desenvolvido utilizando **Spring Boot**; — ✔️ implementado
-   O frontend web deverá ser desenvolvido utilizando **React**; — ✔️ implementado
-   O aplicativo mobile deverá ser desenvolvido utilizando **Flutter**; — ⏳ Sprint 5 ([#36](https://github.com/ICEI-PUC-Minas-PPLES-TI/plf-es-2026-1-ti5-0492100-agrolink/issues/36))
-   O banco de dados utilizado será **MySQL**; — 🔍 [#51](https://github.com/ICEI-PUC-Minas-PPLES-TI/plf-es-2026-1-ti5-0492100-agrolink/issues/51) (dev local ainda usa H2)
-   A comunicação entre cliente e servidor deverá seguir o padrão **RESTful**; — ✔️ implementado
-   O sistema de comunicação em tempo real (chat) deverá utilizar **WebSocket**; — ⏳ polling na web/mobile; WebSocket não implementado ([#38](https://github.com/ICEI-PUC-Minas-PPLES-TI/plf-es-2026-1-ti5-0492100-agrolink/issues/38))
-   A visualização de localização geográfica deverá utilizar **Google Maps API**; — ⏳ Leaflet/OSM no código ([#26](https://github.com/ICEI-PUC-Minas-PPLES-TI/plf-es-2026-1-ti5-0492100-agrolink/issues/26), [#40](https://github.com/ICEI-PUC-Minas-PPLES-TI/plf-es-2026-1-ti5-0492100-agrolink/issues/40))
